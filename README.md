我用 LangChain + B 站搜索 + DeepSeek AI 搭了个「食材→菜谱 + 视频」的全自动助手，输入家里剩的菜，就能一键拿到高赞 B 站做菜视频和保姆级详细菜谱，厨房小白也能秒变大厨！
🧠 核心技术思路（人话版）
整个项目就是一套 「搜→总结→美化」的自动化流水线 ，我用 LangChain 把各个环节串起来，让它们像流水线工人一样配合干活
1. 前端输入 → 后端接收
用户在前端输入食材（比如「鸡蛋、番茄」），通过接口传给后端 FastAPI 服务。
2. LangChain 串联核心能力
我用 LangChain 的 Runnable 链，把「B 站搜索」和「AI 总结」两个步骤焊死在一起：
第一步：B 站搜索：调用 B 站 API，根据食材关键词搜「家常菜教程」，按播放量排序，挑出最火的视频信息（标题、UP 主、链接、播放量）。
第二步：AI 总结：把视频信息喂给 DeepSeek AI，让它生成详细、流畅、排版好看的菜谱步骤，不限制步骤数量，新手也能看懂。
第三步：格式美化：AI 直接输出带 HTML 标签的内容，前端渲染后就是你看到的精致卡片样式。
3. 前后端不分离部署
最后把前端 Vite 打包成 dist 文件夹，塞进 FastAPI 后端，让 Python 直接托管前端页面，只启动一个服务，全世界都能访问，再也不用头疼跨域问题！
